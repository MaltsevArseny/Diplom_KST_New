1. ВВЕДЕНИЕ

1.1. Зачем было создано это приложение

Идея проекта родилась из реальной задачи: небольшие магазины компьютерной
техники часто работают без постоянного интернета (например, точки продаж
в торговых центрах, районных магазинах). Им нужна программа, которая:

  - хранит каталог товаров прямо на компьютере (без серверов и облаков);
  - позволяет покупателям самостоятельно выбирать товары и оформлять заказ;
  - даёт администратору (сотруднику магазина) удобный инструмент для
    управления ассортиментом, заказами и складскими остатками.

Приложение получило название DigitalHub — «цифровой центр» — и работает
целиком на одном компьютере, без подключения к интернету.

1.2. Для кого предназначено

  - Покупатель — обычный клиент магазина. Регистрируется, просматривает
    каталог, добавляет товары в корзину, оформляет заказ.
  - Администратор — сотрудник магазина. Добавляет и редактирует товары,
    обрабатывает заказы, следит за остатками на складе.

1.3. Словарь терминов

  Desktop-приложение — программа, которая устанавливается и запускается
    прямо на компьютере пользователя, в отличие от сайтов, которые
    открываются в браузере.

  JavaFX — набор инструментов (фреймворк) для создания графического
    интерфейса программ на языке Java. Позволяет рисовать кнопки,
    таблицы, формы и т.д.

  SQLite — лёгкая база данных, которая хранит все данные в одном файле
    на жёстком диске. Не требует отдельного сервера.

  CRUD — сокращение от Create (создать), Read (прочитать), Update (обновить),
    Delete (удалить) — четыре базовые операции с данными.

  Хеширование пароля — превращение пароля в набор символов, из которого
    нельзя восстановить исходный пароль. Даже если злоумышленник получит
    доступ к базе, он не узнает настоящие пароли.

  PBKDF2 — алгоритм хеширования паролей. Специально сделан медленным,
    чтобы перебор (brute-force) занимал очень много времени.

  AES-256 — алгоритм шифрования данных. 256 — длина ключа в битах;
    считается одним из самых надёжных на сегодня.

  PII (персональные данные) — информация, по которой можно идентифицировать
    конкретного человека: телефон, адрес доставки и т.д.

  3НФ (третья нормальная форма) — способ проектирования базы данных,
    при котором каждый факт хранится только в одном месте. Это устраняет
    дублирование и снижает риск противоречивых данных.

  Пагинация — разбиение большого списка на «страницы», чтобы не загружать
    сразу все товары и не тормозить интерфейс.

  Паттерн Undo — возможность отменить действие в течение нескольких секунд
    (как в Gmail при удалении письма).

3. ПОДРОБНЫЕ ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

3.1. Регистрация и вход

  Чтобы пользоваться приложением, нужно создать аккаунт. При регистрации
  система проверяет:
    - Имя — от 3 до 50 символов
    - Email — должен быть настоящего формата (ivanov@mail.ru)
    - Телефон — российский формат +7XXXXXXXXXX (11 цифр)
    - Пароль — минимум 8 символов, хотя бы одна заглавная буква,
      одна строчная, одна цифра и один спецсимвол (!@$%^&)

  За безопасность отвечают два механизма:
    1) Пароль не хранится в открытом виде — вместо этого сохраняется
       его хеш (PBKDF2 с 100 000 итерациями). Даже администратор
       не может узнать пароль пользователя.
    2) Защита от перебора: после 5 неудачных попыток входа аккаунт
       автоматически блокируется на 5 минут.

3.2. Личный кабинет

  Пользователь может изменить своё имя, email и телефон. Также доступна
  смена пароля (нужно ввести старый, затем новый дважды).

3.3. Каталог товаров

  Товары сгруппированы по 11 категориям: Процессоры, Видеокарты,
  Материнские платы, Оперативная память, Накопители, Блоки питания,
  Корпуса, Охлаждение, Периферия, Мониторы, Сетевое оборудование.

  Каждая категория имеет свой цвет и эмодзи для быстрого распознавания.
  Категории по умолчанию свёрнуты (аккордеон) — раскрываются по клику.

  Чтобы не загружать интерфейс, в каждой категории сначала показываются
  первые 15 товаров, а остальные подгружаются кнопкой «Показать ещё».

  Поиск работает по названию и описанию. Фильтры: по категории и по цене.

  Прямо из карточки товара можно:
    - Добавить в корзину (кнопка меняется на «✓ В корзине»)
    - Добавить в избранное (сердечко ♡ → ❤)
    - Открыть подробное описание (клик по названию)

3.4. Корзина

  Товары в корзине сгруппированы по категориям. Количество можно менять
  кнопками + и − прямо в списке. Если нажать − при количестве = 1,
  товар предлагается удалить.

  При удалении товара появляется кнопка «Отмена» — есть 5 секунд на
  то, чтобы передумать (паттерн Undo). Кнопка + блокируется, когда
  достигнут складской остаток.

  Внизу корзины — итоговая сумма и кнопка «Оформить заказ».

3.5. Избранное

  Сохранённые товары, сгруппированные по категориям. Из избранного можно
  сразу добавить товар в корзину или удалить (тоже с Undo 5 сек).

3.6. Оформление заказа

  Покупатель указывает адрес доставки, контактный телефон, дату и интервал
  доставки (двухчасовые окна с 08:00 до 23:00), комментарий. При повторном
  заказе форма заполняется автоматически из предыдущего.

  Оплата — при получении (постоплата). Система проверяет остатки на
  складе перед оформлением.

3.7. История заказов

  Все заказы отображаются с группировкой по статусу. Каждый заказ имеет
  наглядный прогресс-трекер:

    ✓ Выполненные шаги — зелёные (с датой и временем)
    ● Текущий этап — фиолетовый, выделенный
    ○ Будущие шаги — серые
    ✕ Отменён — красный

  Статусы заказа: Новый → В обработке → Подтверждён → Собран →
  Отправлен → Доставлен → Завершён. Отмена возможна на любом этапе.

3.8. Панель администратора

  Отдельный интерфейс с четырьмя разделами:
    - Товары: добавление, редактирование, удаление, остатки
    - Заказы: смена статуса, назначение даты доставки
    - Пользователи: блокировка/разблокировка, создание администраторов
    - Отчёты и аналитика (подробнее — в п. 3.9)

3.9. Отчёты и аналитика (панель администратора)

  Администратору доступен раздел аналитики с фильтрацией по датам
  (быстрые кнопки 7д/30д/90д, произвольный период, сброс).
  Отчёт состоит из пяти блоков:

  1) Ключевые показатели (KPI) — четыре карточки:
     суммарная выручка, количество заказов, товаров продано, средний чек.

  2) Конверсия клиентов — сколько зарегистрированных
     пользователей сделали хотя бы один заказ (процент конверсии)
     и сколько из них вернулись за повторной покупкой.

  3) Топ-10 товаров по выручке — гистограмма с названиями,
     категориями и долей в процентах.

  4) Продажи по категориям — гистограмма по 11 категориям
     с выручкой и долями.

  5) Топ-10 по каждой категории — плитки (по 3 в ряд) с
     мини-гистограммами для каждой из 11 категорий.

  Любой отчёт можно распечатать: приложение формирует
  HTML-страницу и открывает её в браузере (Ctrl+P).

3.10. Горячие клавиши

  Для ускорения работы реализованы горячие клавиши:

  Покупатель:                    Администратор:
    Alt+1 — Каталог                Alt+1 — Товары
    Alt+2 — Корзина                Alt+2 — Заказы
    Alt+3 — Избранное              Alt+3 — Пользователи
    Alt+4 — Заказы                 Alt+4 — Отчёты
    Alt+5 — Профиль                Ctrl+P — Печать отчёта
    F1    — Справка                F1    — Справка
    Alt+Q — Выход                  Alt+Q — Выход
    Alt+M — Свернуть               Alt+M — Свернуть
    Alt+W — Закрыть                Alt+W — Закрыть

3.11. Управление окном

  Приложение использует собственный заголовок окна (без стандартной
  рамки Windows). Кнопки: свернуть, развернуть/восстановить, закрыть.
  Окно перетаскивается за верхнюю панель. Двойной клик — развернуть.

3.12. Встроенная справка

  Кнопка «Справка» в навбаре открывает руководство пользователя
  (или руководство администратора) прямо внутри приложения, без
  перехода на другой экран.

4. БАЗА ДАННЫХ

   Тип: SQLite 3 — вся база хранится в одном файле (dist/digitalhub.db).
   Никаких серверов — файл лежит рядом с программой.

   База спроектирована по третьей нормальной форме (3НФ):
     - Справочник категорий вынесен в таблицу Categories (11 записей)
     - Справочник статусов заказов вынесен в таблицу OrderStatuses (8 записей)
     - Таблицы Products, Orders и OrderStatusHistory ссылаются на
       справочники через внешние ключи (category_id, status_id)
     - Это устраняет дублирование строковых значений и обеспечивает
       целостность данных через ссылочные ограничения

   При первом запуске приложение автоматически:
     - Создаёт 9 таблиц (Users, Products, Orders, OrderItems, Cart,
       Favorites, OrderStatusHistory, Categories, OrderStatuses)
     - Заполняет справочники категорий (11 шт.) и статусов (8 шт.)
     - Заполняет каталог 500 товарами по 11 категориям
     - Создаёт 11 тестовых пользователей (8 покупателей и 3 администратора)
     - Генерирует 170 тестовых заказов, равномерно распределённых
       с 01.01.2025 по 01.03.2026, с реалистичной историей статусов,
       чтобы в разделе «Аналитика» сразу были видны графики и статистика

6. АРХИТЕКТУРА СИСТЕМЫ

  Язык программирования: Java 21
  Графический интерфейс:  JavaFX 21
  База данных:            SQLite 3 (один файл, схема в 3НФ)
  Система сборки:         Apache Maven
  Тестирование:           JUnit 5 + JaCoCo

  Архитектурный паттерн: MVVM (Model–View–ViewModel).
  Это значит, что код разделён на три слоя:
    - Model — данные и работа с базой (классы User, Product, Order...)
    - View — то, что видит пользователь (экраны, кнопки, таблицы)
    - ViewModel/Service — логика между ними (валидация, расчёты)

  Такое разделение упрощает тестирование: можно проверить логику
  отдельно от интерфейса.

  Сборка и запуск:
    - maven упаковывает всё в один JAR-файл (dist/DigitalHub.jar)
    - run.bat — пересборка + запуск (Windows)
    - run.sh  — пересборка + запуск (Linux / macOS)
    - dist/DigitalHub.bat — быстрый запуск без пересборки
